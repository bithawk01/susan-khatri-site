Java vs JDK vs JRE
Explained Simply

If you have ever wondered what the difference is between Java, the JDK, and the JRE — and which one you actually need — this guide explains it all in plain language with zero confusing jargon.

// The Short Answer First

Which One Do You Need?

Before the detailed explanation, here is the quick answer most students are looking for.

🎯 Quick Recommendation

✏️

Writing and learning Java? → Download the JDK

If you are a student writing Java programs, your class assignments require compiling code. Only the JDK includes the javac compiler. This is what you need. The JDK also contains the JRE inside it, so you get everything in one download.

▶️

Just running a Java app someone else made? → JRE is enough

If a teacher gives you a pre-compiled .jar file to run and you are not writing any code yourself, the JRE alone is sufficient. It can run Java programs but cannot compile them.

🎓

Taking a CS or programming course? → Always get the JDK

When in doubt, get the JDK. It includes everything the JRE has, plus more. You can never go wrong downloading the JDK — it is the safe choice for any student.

// The Full Explanation

Each Term Explained

Here is what each term actually means, written in plain English.

Java

The Programming Language

// the language you write code in

Java (with a capital J) is a programming language — a set of rules, keywords, and syntax for writing instructions that a computer can follow. When your teacher says "we are learning Java this semester," they mean the language. Java was created by Sun Microsystems in 1995 and is now maintained by Oracle. It is one of the most widely used languages in the world, especially in schools, universities, and enterprise software. Writing Java code means creating .java files full of instructions. But to actually run those instructions, you need additional tools — that is where the JDK and JRE come in.

The programming language syntax and rules
Keywords like class, public, static, void, if, for
Object-oriented programming concepts built into the language
JDK

Java Development Kit

// the full toolbox for developers

The JDK is the complete package for anyone who wants to write Java programs. Think of it as a full professional toolbox. It contains the compiler that turns your .java source files into .class bytecode files, plus the JRE (so you can also run programs), plus extra debugging and profiling tools. When you download Java 21 from Oracle's website, you are downloading the JDK. Students writing Java code for school assignments, projects, or personal learning always need the JDK.

javac — the Java compiler (turns .java into .class)
java — the program launcher (runs compiled code)
jar — tool for packaging Java apps into .jar files
javadoc — automatic documentation generator
The complete JRE (included inside the JDK)
JRE

Java Runtime Environment

// just enough to run Java programs

The JRE is a smaller package that only contains what is needed to run Java programs — it cannot compile them. It is like having a DVD player but no camera. You can play movies (run programs), but you cannot record new ones (write and compile code). In older versions of Java (before Java 11), Oracle distributed the JRE separately as a lighter download. From Java 11 onwards, Oracle stopped distributing a separate JRE. The JDK now serves both purposes. So for modern Java versions, JDK is what you download regardless.

java — the program launcher
The Java standard class libraries
The JVM (Java Virtual Machine)
Everything needed to run compiled .class or .jar files
JVM

Java Virtual Machine

// the engine that actually runs your code

The JVM is the actual engine underneath everything. It reads compiled Java bytecode (the .class files) and executes them. The JVM is what makes Java famous for being "write once, run anywhere" — because different operating systems (Windows, Mac, Linux) each have their own JVM, the same compiled Java program runs on all of them without any changes. You do not interact with the JVM directly. It runs automatically when you type java MyProgram in the terminal. It is included inside both the JRE and the JDK.

Reads and executes Java bytecode (.class files)
Makes Java platform-independent (same code, any OS)
Manages memory automatically (garbage collection)
Included inside both the JRE and JDK

How They Fit Together

The JDK contains the JRE, which contains the JVM. Think of it like nesting dolls.

// FAQ

Frequently Asked Questions

The questions students ask most often about Java, JDK, and JRE.

My teacher said to install Java. Do they mean JDK or JRE?
Almost certainly the JDK. When teachers say "install Java" in a programming course context, they mean the full development kit. If your course involves writing any code at all, you need the JDK. Download Java 21 JDK from Oracle or from the Java 21 guide on this site.
If JDK includes everything, why does JRE exist at all?
Historically, the JRE existed as a smaller, lighter download for people who just needed to run Java applications — not develop them. At one point, many websites used Java plugins that required the JRE. Today, Java web plugins are gone, and Oracle no longer ships a separate JRE for Java 11 and above. For modern Java, just download the JDK and you have everything.
Can I use javac with the portable JDK ZIP?
Yes. The portable Java 21 JDK ZIP from Oracle includes javac inside the bin folder. After setting your PATH, you can compile with javac MyFile.java and run with java MyFile — everything works exactly as it does with an installed version.
What is OpenJDK and is it different from Oracle JDK?
OpenJDK is the open-source version of Java. Oracle JDK is Oracle's official build, which is based on OpenJDK but has some additional features and long-term commercial support. For students, both work identically. Oracle JDK is free for personal and educational use. OpenJDK builds (like those from Adoptium) are always free with no licensing concerns.
